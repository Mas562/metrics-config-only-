# Пороговые значения и обоснование метрики слов-паразитов (filler)

## Выбранный алгоритм

**Алгоритм:** `filler_ratio_penalty_baseline` — штрафная оценка на основе доли слов-паразитов в транскрипте.

Метрика входит в `confidence_index` с весом `0.35` — наибольшим среди всех метрик.

## Определение метрики

Логика метрики:
1. Из транскрипта извлекается список слов-паразитов (fillers) и общее число слов.
2. Если текст пустой — используется `empty_text_fallback_words = 1` для предотвращения деления на ноль.
3. Вычисляется `filler_ratio = len(fillers) / total_words`, округлённый до `filler_ratio_precision = 4` знаков.
4. Если деление невозможно — используется `empty_ratio = 0`.
5. `filler_score = max(0, base_score - filler_ratio * penalty_multiplier)` = `max(0, 100 - ratio * 750)`.

## Параметры

### `empty_text_fallback_words`
- **Значение:** `1`
- **Смысл:** fallback-количество слов при пустом тексте.
- **Обоснование:** предотвращает деление на ноль. При пустом тексте ratio = 0, score = 100 (нет паразитов — нет штрафа).

### `empty_ratio`
- **Значение:** `0`
- **Смысл:** fallback-значение ratio при невозможности вычисления.

### `filler_ratio_precision`
- **Значение:** `4`
- **Смысл:** точность округления `filler_ratio`.
- **Обоснование:** 4 знака после запятой достаточно для точного расчёта штрафа при малых значениях ratio.

### `filler_score.base_score`
- **Значение:** `100`
- **Смысл:** максимальная оценка при отсутствии слов-паразитов.

### `filler_score.penalty_multiplier`
- **Значение:** `750`
- **Смысл:** множитель штрафа за долю слов-паразитов.
- **Обоснование:** при `penalty_multiplier = 750` функция штрафа выглядит так:

| filler_ratio | Доля паразитов | filler_score |
|:-------------|:---------------|:-------------|
| 0.000 | 0% | 100 |
| 0.013 | ~1 из 75 слов | ~90 |
| 0.027 | ~1 из 37 слов | ~80 |
| 0.067 | ~1 из 15 слов | ~50 |
| 0.100 | ~1 из 10 слов | ~25 |
| 0.133 | ~1 из 7 слов | 0 |

Множитель 750 подобран так, чтобы:
- Редкие паразиты (1–2%) давали заметный, но не катастрофический штраф.
- Частые паразиты (>10%) давали score близкий к нулю.
- Полное обнуление score происходило при ratio ~13% (каждое 7-е слово — паразит), что соответствует очень плохой речи.

### `filler_score.min_score`
- **Значение:** `0`
- **Смысл:** нижняя граница `filler_score`.

## Эффективность алгоритма

Метрика filler является детерминированной при наличии корректного транскрипта и списка слов-паразитов. Качество метрики напрямую зависит от:
1. Точности ASR-транскрипции.
2. Полноты словаря слов-паразитов.

**Ограничения:**
1. Метрика не учитывает контекст: слово «ну» в одном контексте — паразит, в другом — союз.
2. Зависит от качества ASR: при ошибках транскрипции паразиты могут не распознаваться.
3. Линейная функция штрафа не учитывает кластеризацию паразитов (несколько подряд воспринимаются хуже, чем равномерно распределённые).

**Возможное улучшение (v2):**
Добавить повышенный штраф за кластеры паразитов (3+ подряд), не меняя основную формулу. Это небольшое дополнение к текущему алгоритму.

## Коэффициенты и веса

Текущий вес в `confidence_index`: `0.35` — наибольший среди всех метрик.

Слова-паразиты — наиболее объективно измеримый и значимый показатель качества речи. Их наличие напрямую снижает воспринимаемую компетентность спикера. Высокий вес обоснован. Подробнее — в `confidence_index/config.md`.
