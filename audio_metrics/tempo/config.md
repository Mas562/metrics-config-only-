# Пороговые значения и обоснование метрики темпа речи

## Выбранный алгоритм

**Алгоритм:** `sliding_window_wpm_baseline` — расчёт темпа речи через скользящее окно по временным меткам слов.

Метрика темпа не входит в `confidence_index` как отдельный score, но используется для формирования детальной обратной связи: временной ряд WPM с разметкой по зонам (red/yellow/green).

## Определение метрики

Логика метрики:
1. На вход поступает транскрипт с временными метками слов.
2. По временной шкале строятся точки с шагом `step_sec = 1.0` сек.
3. Для каждой точки считается количество слов в окне `window_sec = 5.0` сек вокруг неё.
4. Количество слов умножается на `wpm_multiplier = 60` и делится на `window_sec`, получая WPM.
5. Значение WPM округляется до `wpm_round_digits = 1` знака.
6. Каждая точка окрашивается в зону: red / yellow / green по правилам из `zones`.

## Параметры

### `window_sec`
- **Значение:** `5.0` сек
- **Смысл:** размер скользящего окна для подсчёта слов.
- **Обоснование:** окно 5 сек достаточно для сглаживания локальных флуктуаций темпа (вдох, пауза) и при этом достаточно коротко, чтобы отражать реальные изменения темпа в ходе выступления.

### `step_sec`
- **Значение:** `1.0` сек
- **Смысл:** шаг по временной шкале между точками WPM.
- **Обоснование:** шаг 1 сек даёт достаточную детализацию для визуализации темпа без избыточного количества точек.

### `wpm_multiplier`
- **Значение:** `60`
- **Смысл:** коэффициент перевода числа слов в окне в words per minute.
- **Обоснование:** стандартная единица измерения темпа речи — WPM. Множитель 60 / window_sec переводит слова-в-окне в WPM.

### `wpm_round_digits`
- **Значение:** `1`
- **Смысл:** точность округления WPM.
- **Обоснование:** точность до 0.1 WPM достаточна для отображения и анализа.

### Зоны темпа

| Зона | Диапазон WPM | Интерпретация |
|:-----|:-------------|:--------------|
| **green** | 100–140 | Оптимальный темп для публичного выступления |
| **yellow** | 80–100 или 140–160 | Допустимый, но требующий внимания темп |
| **red** | < 80 или > 160 | Слишком медленно или слишком быстро |

#### Обоснование зон

Диапазон 100–140 WPM — общепринятый стандарт для публичных выступлений и презентаций:
- Ниже 80 WPM: речь воспринимается как монотонная и затянутая.
- 80–100 WPM: медленновато, но допустимо для сложного технического контента.
- 100–140 WPM: оптимальный диапазон — аудитория успевает воспринимать информацию.
- 140–160 WPM: быстровато, но допустимо для энергичных спикеров.
- Выше 160 WPM: аудитория не успевает усваивать информацию.

Источник ориентиров: исследования восприятия публичной речи (National Communication Association, данные TED Talks — средний темп ~163 WPM, оптимальный для восприятия — 120–150 WPM).

## Эффективность алгоритма

Метрика темпа является детерминированной при наличии точных временных меток ASR. Алгоритм скользящего окна — стандартный и хорошо изученный подход.

**Ключевой критерий качества:** точность временных меток ASR и корректность токенизации слов.

**Ограничения:**
- Метрика не учитывает сложность контента: быстрый темп при простом тексте может быть нормой, а медленный при сложном — необходимостью.
- Скользящее окно сглаживает резкие переходы темпа.
- Метрика не входит в `confidence_index` напрямую — при необходимости можно добавить производный score (доля времени в зелёной зоне).

## Коэффициенты и веса

Метрика темпа не входит в `confidence_index` напрямую. Используется как качественная обратная связь с временным рядом WPM и разметкой по зонам.
