# Шаблон для оценки метрик жестикуляции

Эта папка — заготовка, которую можно перенести в другой проект и использовать для сравнения нескольких способов оценки жестикуляции на заранее подготовленной выборке видео.

## Структура

- `media/` — исходные ролики для анализа
- `results/` — промежуточные результаты, выгрузки, графики и итоговые таблицы
- `notebooks/gesture_metrics_comparison.ipynb` — основной ноутбук для запуска сравнения методов
- `dataset_manifest.csv` — реестр всех роликов, вошедших в выборку
- `comparison_table.csv` — итоговая сравнительная таблица по методам
- `config.json` — пороги и коэффициенты для выбранного лучшего алгоритма
- `config.md` — обоснование выбора алгоритма, порогов и коэффициентов

## Какая выборка ожидается

Нужно подготовить три группы роликов:

1. `good_professional` — 4 профессиональных ролика с естественной, активной, но не чрезмерной жестикуляцией.
2. `bad` — 2 ролика со слабой, скованной или неудачной жестикуляцией.
3. `self_recorded` — собственные короткие ролики по 3–5 минут, записанные на разном расстоянии от камеры (`near`, `mid`, `far`).

Для собственных роликов:
- стараться жестикулировать профессионально,
- 2–3 раза показать на воображаемую доску или слайды,
- по возможности держать схожую манеру речи на всех дистанциях, чтобы проверить устойчивость метрики к расстоянию.

## Какие методы обязательно сравнивать

Ноутбук подготовлен как минимум под три метода:

1. **Текущий baseline из Charisma Master**
   - Основан на текущей реализации `gesture_score`.
   - Референсная логика находится в `charisma-master/services/ml_worker/app/logic/ml_engine.py`.
   - Связанные параметры лежат в `charisma-master/services/ml_worker/app/config.json`.

2. **Простой метод, предложенный нейросетью**
   - Более эффективный, но без лишней сложности.
   - Предполагается использование нормализованных признаков движения, амплитуды и доли активной жестикуляции.

3. **Более сложный, но CPU-friendly метод**
   - Более содержательный по признакам, но всё ещё пригодный для обычного CPU.
   - Предполагается использование эпизодов жестикуляции, ритма и вариативности движений.

## Что должно получиться в итоге

В результате должны быть готовы:
- заполненный `dataset_manifest.csv`,
- заполненный `comparison_table.csv`,
- ноутбук с реализацией и сравнением всех трёх методов,
- выбранные пороги в `config.json`,
- текстовое обоснование в `config.md`.

## Что сделать перед коммитом

Перед коммитом нужно очистить outputs у Jupyter notebook, чтобы не тянуть в репозиторий лишний бинарный мусор.
