# Gaze Metrics Config Fields

Этот файл описывает поля `config.json` для модуля анализа взгляда (**video_metrics/gaze/**). Эти параметры управляют логикой определения зрительного контакта на основе комбинации поворота головы (Head Pose) и направления зрачков (Iris Tracking).

## Общая идея

В отличие от базового метода, Gaze V2 учитывает, что спикер может смотреть в камеру, даже если его голова повернута (логика компенсации). Конфиг позволяет настраивать «мертвые зоны» и чувствительность трекинга зрачков без изменения программного кода.

---

## `head_pose`
Параметры ориентации головы в пространстве, вычисляемые через задачу PnP (Perspective-n-Point).

### `yaw_center_offset`
- **Что делает:** задаёт калибровочное смещение для угла Yaw (поворот влево-вправо). 
- **Зачем это нужно:** в используемой модели MediaPipe PnP за «ноль» (взгляд в камеру) принимается значение 180 градусов. Это поле нормализует угол в диапазон, где 0 — это центр.

### `pitch_center_offset`
- **Что делает:** задаёт калибровочное смещение для угла Pitch (наклон вверх-вниз).
- **Зачем это нужно:** на основе тестов выявлено, что при естественной посадке перед камерой средний угол наклона составляет ~10 градусов.

### `yaw_limit`
- **Что делает:** максимальный угол поворота головы (в градусах), при котором анализ взгляда считается валидным и не отбраковывается.
- **Обоснование:** При углах поворота больше этого значения (например, 40 градусов) человек очевидно отворачивается от камеры.

### `pitch_limit`
- **Что делает:** массив из двух значений `[min, max]`, определяющий допустимый наклон головы по вертикали.
- **Зачем это нужно:** позволяет отсекать кейсы «чтения по бумажке» (когда Pitch уходит ниже -20) или чрезмерного задирания головы вверх.

## `iris_tracking`
Параметры отслеживания положения зрачка относительно разреза глаз. Теперь используют динамическую компенсацию с линейной зависимостью.

### `base_gaze`
- **Что делает:** базовая позиция зрачка при фронтальном взгляде в камеру (при yaw = 0).
- **Как работает:** значение 0.5 соответствует точному центру глаза.

### `yaw_gaze_coefficient`
- **Что делает:** коэффициент линейной зависимости смещения зрачка в зависимости от угла поворота головы (yaw).
- **Суть:** на каждый 1 градус поворота головы зрачок смещается примерно на 0.013 (в долях от ширины глаза) в противоположную сторону, чтобы зрительный контакт сохранялся.

### `gaze_tolerance`
- **Что делает:** допустимая погрешность отклонения взгляда (в долях).
- **Обоснование:** допускает микро-движения глаз или неточность трекинга. Значение 0.08 дает окно прощения в ±8% от ширины глаза вокруг ожидаемой `expected_gaze` позиции.

## `landmarks_v2`
Специфические индексы точек Face Mesh для Gaze V2.

### `pnp_indices`
- **Что делает:** список из 6 индексов (нос, подбородок, углы глаз и рта), используемых для построения 3D-проекции головы.
- **Где используется:** `get_head_pose_v2()`.

### `iris_indices_left` / `iris_indices_right`
- **Что делает:** индексы точек, ограничивающих зрачок (474-477 и 469-472).
- **Где используется:** `get_gaze_ratio()`.

---

## `processing`
Параметры производительности.

### `sampling_rate`
- **Что делает:** определяет частоту анализа кадров (например, каждый 5-й кадр).
- **Зачем это нужно:** баланс между точностью и нагрузкой на CPU/GPU воркера.

### `downscale_width`
- **Что делает:** ширина кадра, до которой уменьшается видео перед обработкой в MediaPipe.
- **Обоснование:** 480px достаточно для точной детекции зрачков, но значительно ускоряет работу по сравнению с Full HD.
